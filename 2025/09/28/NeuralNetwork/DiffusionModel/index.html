逐层剖析 Stable Diffusion

  • 从宏观角度看，Stable Diffusion 主要包含两部分：一个文本解释器，一个图像生成器，用户向文本解释器中输入一串文本，经过文本解释器将其投影到特定的语义空间，随后传递给图像生成器中引导图像生成，最终输出一张生成图像。
  • 前面提到的文本解释器本质就是一个 Transformer Encoder，将文本中的词 token 投影到语义空间，得到对应的 embedding。
  • 更进一步，图像生成器主要包含两个阶段：
    • Image Information Creator：这部分是 Stable Diffusion 性能超越其他模型的关键。这部分通常会执行若干步来生成图像信息，执行的步数作为可调参数，通常设置为 50 或 100。
    • Image Decoder：这部分会根据 Image Information Creator 传递来的图像信息绘制图像，只在最后阶段执行一次。
  • 上述三个模块就是 Stable Diffusion 的主要构成，具体如下：
    • Text Encoder：CLIP Text Encoder
      • input：一串文本
      • output：若个干 token embedding 向量，每个向量 768 维
    • Image Information Creator：U-Net + Scheduler
      • input：文本 embedding 向量、一张初始噪声图像
      • output：处理后的噪声图像信息
    • Image Decoder：Autoencoder Decoder
      • input：处理后的噪声图像信息
      • output：生成图像

How Diffusion works

  • 扩散模型生成图像的本质就是预测噪声，通过对原始噪声图像进行逐步去噪得到生成图像。

噪声预测


  • 对于训练集，我们可以通过对不同图片加噪声得到，我们假设 T = 0 时（图中 ammount = 0），图像为原始图像，T = t 时，图像经过了 t 轮加噪，每次加噪都是对原始噪声进行随机采样（实际训练时，为了保证高效，我们只对原始噪声采样一次，加噪时噪声项会乘一个系数，该系数与时刻 T 有关，具体推导过程见参考文章 2）。
  • 有了这个训练集，我们就可以训练一个网络（U-Net）来预测噪声，输入加噪后的图像和时刻 t，输出预测噪声，计算预测噪声与原始噪声的 loss，并反向传播更新网络，最终得到一个功能强大的噪声预测器。

去噪绘制

  • 有了上面训练好的噪声预测器，我们就可以从一个噪声图像经过 T 步去噪，逐步接近最终生成的图像。但是这样生成的图像无法通过文本控制，其完全取决于训练集的图像，由训练集图像的各种特征组合而成。

Diffusion on Latent Space

  • 为了加速图像的生成过程，Stable Diffusion 并没有选择直接在像素空间进行生成，而是先投影到 Latent Space 再进行扩散，相当于生成图像的压缩版本。
  • 这种压缩通过一个 autoencoder 实现，待去噪图像输入后会先被 autoencoder 投影到 latent space，在 latent sapce 去噪后，再通过 autoencoder 的 decoder 投影回像素空间，得到生成图像。这个 decoder 就是前文所提到的 Image Decoder。

文本控制

  • 为了能够让文本控制图像生成过程，我们还需要向 Denoising U-Net 中输入文本信息，输入前需要通过 CLIP Text Encoder 转换为 embedding 向量。

  • 未引入文本控制的 Denoising U-Net 大致结构如图所示，其由若干层 ResNet 组成，每个 ResNet 的输出都是对应时刻去噪后的结果。

  • 为了引入跨模态的文本控制，很自然地想到可以在每个 ResNet 层后加入一个 Cross-Attention 层，由图像隐空间表示来提供 Query 向量（图像特征），文本信息提供 Key 和 Value。需要注意的是，原先的残差连接是从每个 ResNet 的输出直接连接到最后，加入了 Attention 层后，对应时刻的去噪结果是 Attention 层的输出，因此残差连接需要从每一个 Attention 层的输出开始。
