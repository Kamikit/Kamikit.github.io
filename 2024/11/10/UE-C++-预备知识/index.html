Lambda表达式（C++11）

  • 允许在代码中定义匿名函数，适合用于定义一些简单的函数，而不必事先声明。

    • 用于指定Lambda表达式可以访问的外部变量，但只限于在Lambda表达式定义位置之前出现过的变量。

      • = ：值捕获Lambda表达式中使用的所有外部变量

Smart Pointer（C++11）

  • 自 C++11 起，标注库引入了唯一指针（unique_ptr）、共享指针（shared_ptr）、弱指针（weak_ptr）。
  • UE则是自己实现了一套智能指针，同时还添加了共享引用，该引用的行为与不可为空的共享指针相同。

唯一指针（TUniquePtr、unique_ptr）：

  • 独占资源的所有权，同一时间只能有一个唯一指针指向给定的资源。
  • 不允许拷贝构造（不允许直接对指针进行拷贝）。
  • 支持移动语义 move，将资源的所有权转移到另一个指针。
  • 唯一指针超出作用域时会自动释放内存。

共享指针（TSharedPtr、shared_ptr）：

  • 用于共享所有权的内存管理，多个共享指针可以指向同一个对象。
  • 采用引用计数，当最后一个共享指针释放时，该对象的内存才会被释放。
  • 共享指针可以为空，意为不引用任何对象。

弱指针（TWeakPtr、weak_ptr）：

  • 与共享指针配合使用，主要用于解决共享指针的循环引用问题。
  • 弱指针不会增加引用计数，因此可以打破循环引用：如果两个共享指针互相引用，那么它们的引用计数永远不会为零，也就无法释放内存。
  • 只可以从一个 shared_ptr 或另一个 weak_ptr 对象构造，构造和析构均不会引起引用计数的变化。
  • 与 shared_ptr 不同，weak_ptr 不能直接访问所指向的对象，要临时访问对象，需要先调用 lock() 方法将其转换为 shared_ptr，如果所指向的对象已经被销毁，则该方法返回空指针。

共享引用（TSharedRef）：

  • 行为与共享指针类似，但共享引用必须固定引用非空对象。
  • 共享引用可以转换为共享指针，且该共享指针固定引用有效对象。
  • 要确认引用的对象是非空，或者要表明共享对象的所有权时，最好使用共享引用

UE智能指针：

注意事项：
  • 应避免使用普通指针变量来初始化智能指针，若不得已为之，则必须将原来的普通指针置 nullptr 且不可 delete，使得只有智能指针指向这块内存
  • 智能指针没有重载加减运算和自加自减运算
  • UE引擎的 UObject 类自带垃圾回收（GC）功能，因此智能指针不能指向 UObject 类的实例，否则可能会导致内存被重复释放
  • 除了 TSharedRef 之外，其他类型的指针都可以使用 Reset() 接口来重置指针。需要注意的是，重置共享指针，会影响到所有指向它的弱指针：
  • 智能指针可以通过接口 Get() 获取原始指针
  • 智能指针的线程安全是可选的，可以通过模板参数控制，参数类型是 ESPMode 枚举类，默认线程不安全从而使性能更佳：
  • 在可以使用 TSharedPtr 和 TSharedRef 时，优先选 TSharedRef
与 C++ 不同之处：
  • 对于 TWeakPtr，必须使用接口 Pin() 来获取其 TSharedPtr，与 lock() 类似
  • 线程安全成为可选项
  • 并未实现一些 C++ 支持的通用接口，如 use_count()、unique() 等
  • UE的智能指针不允许抛出异常
  • 如果 TSharedRef 在创建或赋值时引用空对象，则会触发断言

    • ConstCastSharedRef和ConstCastSharedPtr：用于将 const 类型的智能指针转换为非 const 类型，转换后两个指针指向的是同一块内存，不会申请新内存。
